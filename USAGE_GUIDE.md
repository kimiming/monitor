## 📖 系统使用流程

### 第一步：启动服务

```bash
python run.py
```

服务会在 **http://localhost:5000** 启动

### 第二步：打开管理面板

在浏览器中访问：**http://localhost:5000**

### 第三步：登录账号

#### 3.1 登录监控号

1. 点击"账户"标签页
2. 点击"🔐 登录监控号"按钮
3. **等待 30-120 秒**，终端会显示登录二维码链接或提示
4. 用手机扫码登录（Telegram APP）
5. 登录成功后，前端会显示 ✅

**常见问题：**

- ⏳ **超时（120秒）**：说明网络问题，检查是否需要代理
- 📱 **看不到二维码**：检查终端输出或使用代理重试

#### 3.2 创建克隆号

1. 在"克隆号"部分，输入会话名（如：`sender_1`）
2. 点击"🔐 创建并登录"
3. **等待 30-120 秒**
4. 扫码登录（必须是**不同的 Telegram 账号**）
5. 重复此步骤创建多个克隆号

### 第四步：配置参数

进入"配置"标签页，设置：

| 参数         | 说明                             | 例子                 |
| ------------ | -------------------------------- | -------------------- |
| **API ID**   | Telegram App API ID              | `6`                  |
| **API Hash** | Telegram App API Hash            | `eb06d4ab...`        |
| **源群组**   | 要监控的群组（逗号分隔）         | `@group1, @group2`   |
| **目标群组** | 转发到哪个群（逗号分隔）         | `@target1, @target2` |
| **使用代理** | 是否用手动代理（false=系统代理） | `false`              |
| **代理地址** | SOCKS5 代理地址                  | `127.0.0.1`          |
| **代理端口** | SOCKS5 代理端口                  | `7890`               |

### 第五步：启动转发脚本

1. 回到"仪表板"
2. 点击"▶️ 启动脚本"
3. 脚本开始运行，等待接收消息

## 🔧 代理配置

### 场景 1：使用本地系统代理（推荐）

**配置：** `use_proxy: false`

系统会自动使用：

- Windows 系统代理设置
- 环境变量中的代理配置
- 本地代理工具（Clash、V2Ray 等）

**无需额外配置！** 只需确保本地代理已启动。

### 场景 2：手动指定 SOCKS5 代理

**配置：** `use_proxy: true`

然后设置：

- `proxy_host: 127.0.0.1`
- `proxy_port: 7890`（改成你的代理端口）
- `proxy_username`: 可选
- `proxy_password`: 可选

## 🎯 实际使用示例

### 假设你有：

- 1 个监控号（接收源群消息）
- 3 个克隆号（转发消息到目标群）
- 本地运行 Clash（SOCKS5 在 127.0.0.1:7890）
- 要监控 5 个源群，转发到 2 个目标群

### 步骤：

1. **启动 Clash**（确保已开启）

2. **启动系统**

   ```bash
   python run.py
   ```

3. **打开管理面板**

   ```
   http://localhost:5000
   ```

4. **登录账号**
   - 登录监控号
   - 创建 sender_1, sender_2, sender_3

5. **配置参数**
   - API ID / Hash：填入你的
   - 源群组：`@group1, @group2, @group3, @group4, @group5`
   - 目标群组：`@target1, @target2`
   - 代理：`use_proxy: false`（使用 Clash）

6. **启动脚本**
   - 点击"▶️ 启动脚本"

7. **监控日志**
   - 进入"日志"标签页查看实时日志
   - 查看统计数据

## 📊 监控

### 仪表板

- 系统状态（运行中/已停止）
- 监控号状态
- 克隆号列表及状态
- 消息队列大小

### 日志

- 实时日志显示
- 按级别筛选（INFO, ERROR, WARNING）
- 支持下载日志文件

### 统计

- 过去 N 天的消息统计
- 按克隆号统计
- 按日期统计

## 🛑 停止脚本

1. 点击"⏹️ 停止脚本"按钮
2. 或按 Ctrl+C 停止整个服务

## ⚠️ 常见问题

**Q: 登录时卡住？**
A: 增加超时时间或检查代理是否配置正确

**Q: 克隆号无法登录到目标群？**
A: 需要先手动邀请克隆号进入目标群，或设置为公开群

**Q: 消息转发很慢？**
A: 检查消息队列大小、克隆号数量、系统配置中的并发数

**Q: 出现"TimeoutError"？**
A: 网络不稳定，配置代理后重试
