## 代理配置指南

如果你在中国或其他无法直接连接 Telegram 的地方，需要配置代理。

### 方案 1：使用 SOCKS5 代理（推荐）

1. **安装代理工具**（选一个）：
   - Clash for Windows
   - V2RayN
   - Shadowsocks
   - 等其他支持 SOCKS5 的代理工具

2. **配置本地 SOCKS5 服务**：
   - 大多数代理工具都可以配置本地 SOCKS5 端口（默认通常是 1080 或 7890）

3. **在管理面板配置代理**：
   - 进入"配置"标签页
   - 找到"代理设置"
   - 启用代理：`✓ 使用代理`
   - 代理类型：SOCKS5
   - 代理地址：127.0.0.1
   - 代理端口：1080（根据你的代理工具实际端口）
   - 用户名和密码：如果代理需要认证填写，否则留空

4. **点击保存**

5. **重新登录账号**

### 方案 2：使用 MTProto 代理

Telegram 官方支持 MTProto 代理，但需要知道代理地址和端口。

### 方案 3：使用 Telethon 的自动连接重试

系统已配置自动重试，但需要至少一次成功连接。

## 快速测试代理

如果不确定代理是否工作，可以在命令行测试：

```bash
# 测试 SOCKS5 连接
curl -x socks5://127.0.0.1:1080 https://telegram.org
```

如果返回 HTTP 200 说明代理工作正常。

## 常见问题

**Q：我没有代理怎么办？**
A：可以：

1. 使用免费 VPN（不推荐，不稳定）
2. 购买商业 VPN 服务
3. 搭建自己的代理（技术要求高）

**Q：能用 HTTP 代理吗？**
A：Telethon 不支持 HTTP 代理，必须是 SOCKS5、SOCKS4 或其他协议。

**Q：多少延迟是正常的？**
A：通常 100-300ms 是正常的，如果超过 5 秒会超时。
